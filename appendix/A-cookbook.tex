\chapter{Command cookbook}
\label{app:cookbook}

The reference implementation of the \textsc{swhid} standard (ISO/IEC~18670) is
\texttt{swhid-rs}, a Rust tool released under the MIT licence; use it as the primary local
tool. The Python \texttt{swh identify} (from \texttt{swh.model}) and \texttt{swh scanner} remain
useful for a few operations the Rust CLI does not yet cover --- chiefly snapshot identifiers and
scanning a tree against the archive.

\begin{verbatim}
# --- Install -------------------------------------------------------------
cargo install swhid                  # swhid-rs (Rust reference impl., MIT)
cargo install swhid --features git   # ...with VCS (revision) support
# or download a pre-built binary from the project's releases page:
#   https://github.com/swhid/swhid-rs
pip install 'swh.model[cli]'         # swh identify (Python)
pip install swh.scanner              # swh scanner  (Python)

# --- Compute core SWHIDs locally (Rust reference tool) -------------------
swhid content --file file.c          # -> swh:1:cnt:...
swhid dir src/                       # -> swh:1:dir:...  (Merkle hash of tree)
swhid parse "swh:1:cnt:<hex>"        # validate / pretty-print a SWHID

# A revision SWHID is just the git commit hash:
echo "swh:1:rev:$(git rev-parse HEAD)"     # -> swh:1:rev:...

# --- Verify bytes against a claimed SWHID (exit 0 iff they match) --------
swhid verify file.c swh:1:cnt:<hex>        # PATH first, then SWHID
swhid verify src/   swh:1:dir:<hex>

# --- Operations still served by the Python tools ------------------------
swh identify --type snapshot repo.git/     # -> swh:1:snp:... (snapshot)
swh scanner scan ./repo                    # what is already archived?

# Archive a public repo:  https://save.softwareheritage.org
# Resolve any SWHID:      https://archive.softwareheritage.org/<swhid>
\end{verbatim}

\noindent\textbf{One caveat for \texttt{dir} hashes.} A directory \textsc{swhid} depends on file
\emph{permissions} as well as content, so a tree's hash can differ between a fresh filesystem
checkout and the git index. \texttt{swhid dir}/\texttt{verify} expose
\texttt{-{}-permissions-source} (\texttt{auto}, \texttt{fs}, \texttt{git-index}, \texttt{git-tree},
\dots); to match what the archive stored from a git repository, compute from the git tree (e.g.\
over a clean \texttt{git archive} export, as the Colophon capstone does). If a directory check
mismatches where a content check would pass, suspect permission bits, not the bytes.
